# Code of Conduct

## Our commitment

This project exists to support consensual power exchange between adults.
We welcome contributors of any background, identity, orientation,
relationship structure, or kink practice, as long as their conduct in
project spaces is respectful and focused on the work.

## Expected behavior

- Be respectful in issues, pull requests, commit messages, and
  discussions
- Assume good faith; disagree about technical choices without attacking
  the person making them
- Respect people's stated names, pronouns, and identities
- Keep project spaces work-focused — this is a technical project, not a
  personal ad
- Accept constructive feedback; give it kindly
- Prioritize the safety of users of this software in every design
  decision

## Unacceptable behavior

- Personal attacks, insults, or sustained hostility
- Discriminatory language or conduct of any kind
- Unwanted personal attention, including of a romantic or sexual nature
- Publishing others' private information without explicit permission
  (doxxing)
- Advocating for non-consensual use of this software
- Any conduct that would reasonably make a contributor feel unsafe in
  project spaces

## Power-exchange context

This project models a power-exchange dynamic in its code (Lion,
Bunny, Collar). That vocabulary is a **technical domain model**, not an
invitation to enact a dynamic with other contributors. Project spaces
are egalitarian. Nobody here is anyone else's Lion or Bunny.

## Scope

This Code of Conduct applies to all project spaces: this repository,
its issues and pull requests, any associated chat or forum, and public
events where someone represents the project.

It applies to maintainers, contributors, and users equally.

## Enforcement

Report concerns privately via a GitHub private vulnerability report or
the contact listed in the repository metadata. Reports will be handled
confidentially.

Responses may include: a private warning, a temporary or permanent ban
from project spaces, removal of contributions, or no action if the
report is determined to be unfounded. Repeat or severe incidents
escalate quickly.

Maintainers who violate this Code of Conduct are held to the same
standard as anyone else.

## Safe harbor for good-faith reports

Reporting in good faith will not result in retaliation. Reports made in
bad faith — to harass a contributor, shut down technical disagreement,
or settle a personal grievance — are themselves violations.

## Questions

Open a non-sensitive question in discussions. For anything sensitive,
use the private contact channel.
