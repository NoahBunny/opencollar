What the Relay Sees

  • mesh_id — which mesh a blob belongs to
  • version — monotonic counter per blob
  • ciphertext size — how large each encrypted blob is
  • timing — when blobs are written and read
  • node_id / node_type — device names and roles
  • RSA signatures — verified but not decrypted

This is metadata. The relay uses it for signature verification, version ordering, and storage quotas.

What the Relay Cannot See

  • Lock commands, unlock requests, timer values
  • Paywall amounts, subscription tiers, payment events
  • Messages between Lion and Bunny
  • GPS coordinates, geofence boundaries
  • Photo task images, Lovense commands
  • Any decrypted order content whatsoever

The relay has no decryption keys. Each blob is encrypted with AES-256-GCM, and the AES key is wrapped per-recipient with RSA-OAEP. Only approved vault nodes (Lion's phone, Bunny's phone, approved desktops) hold the private keys needed to unwrap.

How to Verify

The relay is open-source. You can verify that the deployed code matches the published source at any time:

1. curl https://relay-url/version
   # Note the git_commit and source_sha256

2. git clone https://github.com/NoahBunny/openleash
   git checkout <git_commit>

3. sha256sum focuslock-mail.py
   # Compare to source_sha256 from step 1

4. If they match, the relay is running
   unmodified published code.

The /version endpoint is unauthenticated and always available. No trust required — verify it yourself.

Cryptographic Design

  • Body encryption: AES-256-GCM with a fresh random key per blob
  • Key wrapping: RSA-OAEP-SHA256 per recipient (each vault node gets its own encrypted copy of the AES key)
  • Signatures: RSA-PKCS1v15-SHA256 over canonical JSON (sorted keys, compact separators)
  • Slot addressing: SHA-256 of recipient's public key, truncated to 12 hex chars

Full specification: VAULT-DESIGN.md

Self-Host or Go P2P

Don't trust this relay? Two alternatives:

Self-host: The relay is a single Python file with one dependency (cryptography). Runs on any Linux box, Raspberry Pi, or $5/mo VPS.

bash installers/homelab-setup.sh

Peer-to-peer: Use Syncthing transport to sync vault blobs directly between devices. Zero relay, zero VPS, fully P2P. Set "vault_transport": "syncthing" in your config.

Your Lion's Share app connects to your own infrastructure. Same encryption, same protocol, your hardware. Switch any time — no lock-in.
